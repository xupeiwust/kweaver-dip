# KWeaver DIP 项目

你的任务是开发 KWeaver DIP 项目

## 项目结构

```
.
├── .github/
│   └── workflows/        # CI/CD 流水线
├── deploy/               # 部署脚本及配置
├── docs/                 # 项目文档
├── design/               # 特性设计及交互设计
├── release-notes/        # 版本发布说明
├── web/                  # 前端代码
├── studio/               # 数字员工平台模块
│   └── AGENTS.md         # 数字员工后端服务开发指南
├── dsg/                  # 数据语义治理模块
├── chat-data/            # “数据分析员”模块
├── hub/                  # 应用商店模块
└── skills/               # AI Agent 技能
```

## 开发流程

1. 首先识别用户需求，根据项目结构确定要读取的模块以及要开发的模块：
  - 如果设计功能逻辑或交互设计，优先读取 `design/` 目录下的特性及交互设计文档，从 `design/INDEX.md` 了解详细的子目录结构。
2. 在所需开发功能模块目录下进一步查找指南文档，按以下优先级进行检索：
  - INDEX.md
  - AGENTS.md
3. 如果在功能模块目录下找到指南文档，仔细阅读内容。如果指南中还有进一步被引用的文档，则跟随索引路线进一步检索并阅读。
