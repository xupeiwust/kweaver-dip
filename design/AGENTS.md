# 特性设计及交互设计

## 目录结构

本目录存放特性设计及交互设计，目录结构如下：

```
.
│   data-model/           # 数据库设计
│   └── studio/           # 数字员工平台数据库设计
├── feature/              # 特性设计
│   └── studio/           # 数字员工平台特性设计
│   └── hub/              # AI 应用商店特性设计
│   └── web/              # 通用的 Web 设计
├── ux/                   # 交互设计目录
│   └── studio/           # 数字员工平台交互设计
```

## 读取规则

1. 首先根据用户需求，从 feature/ 目录下检索文档元数据，找到对应的特性设计文档。
2. 仔细阅读特性设计文档，充分理解业务需求。
3. 从 ux/ 目录下读取读取文档元数据，找到对应的交互设计文档。
4. 仔细阅读交互设计文档。如果交互设计文档中包含 Figma 设计稿，通过 Figma MCP 读取设计稿，并结合交互设计文档开发前端代码。
