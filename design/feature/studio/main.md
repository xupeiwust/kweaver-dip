# 第一部分 总体设计

本文档关于 DIP Studio 平台上定义和运维数字员工，实现管理视角的 human-on-the-loop。

# 第二部分 逻辑设计

## 一、术语

**数字员工**

数字员工是配置了领域专用知识和技能，具备推理、规划、执行能力，24/7 在线的企业数字助理。

**技能（Skill）**

技能（Skill）是一组面向智能体的指令描述，用于教会智能体如何执行特定的任务或流程。

**通道（Channel）**

通道（Channel）是用户与数字员工进行对话的应用入口，一般是各类即时通信工具（如：飞书、钉钉、企业微信），DIP Studio 提供的对话入口也是通道之一。

## 二、角色定义

| 角色 | 职责 |
| --- | --- |
| 普通用户 | 在通道中（IM + DIP Studio）与数字员工进行协作<br>- 管理自己下发给数字员工的计划和任务 |
| 管理员 | 创建、管理数字员工<br>- 配置数字员工通道（Channel）<br>- 管理数字员工技能（Skill）<br>- 管理领域知识网络（BKN） |

## 三、首页

### 3.1、主对话框

1. 具有普通用户角色的用户可以在首页使用主对话框与数字员工进行对话。
2. 支持通过 @数字员工 与指定的数字员工对话。
3. 在主对话框发送消息后，会发起一次新的会话，会话成员包括用户 + 所有权限范围内的数字员工。
4. 用户可以在主对话框或会话界面展开侧边栏加载历史会话。

### 3.2、数字员工列表

1. 数字员工以卡片的方式呈现。
2. 数字员工的可见性取决于用户角色和用户对数字员工的访问权限：
  - 管理员可以查看所有数字员工
  - 普通用户仅可查看被授权的数字员工
3. 数字员工包含以下基本属性：  
  | 属性 | 说明 |
  | --- | --- |
  | 名称 |  |
  | 简介 |  |
  | 创建者 | 不可编辑。 |
  | 创建时间 | 不可编辑。 |
  | 编辑者 | 不可编辑。创建新数字员工时，编辑者 = 创建者 |
  | 编辑时间 |不可编辑。 创建新数字员工时，编辑时间 = 创建时间 |
4. 卡片除了展示部分基本属性外，还显示以下统计数据：
| 数据 | 说明 | 形式 |
| --- | --- | --- |
| 使用者数量 | 展示与数字员工协作的用户数量：<br>- 按 “用户ID + 数字员工ID” 去重计数<br>- 仅对管理员展示  | 用户头像，超过 3 个在末尾显示数量  |
| 计划 | 展示数字员工的计划数量：<br>- 对管理员展示总计划数量<br>- 对普通用户展示本人创建的计划数量 | 数字 |
| 任务 | 展示数字员工最近 7 天执行的成功率趋势：<br>- 对管理员展示全部任务的统计结果<br>- 对普通用户展示本人创建的计划下的任务的统计结果 | 折线图（迷你、极简） |
5. 卡片支持的操作包括：
  - 编辑
  - 删除
6. 点击卡片进入数字员工详情，可以对数字员工进行定义和运维。

### 3.3、新建数字员工

1. 新建数字员工时需要填写以下属性：

| 属性 | 是否必填 | 备注 |
| --- | --- | --- |
| 名称 | 必填 | 最多 128 个字符 |
| 简介 | 非必填 | 最多 400 个字符 |

2. 新建数字员工成功后进入数字员工详情。

### 3.4 删除数字员工

1. 删除数字员工时，用户需要输入数字员工名称进行二次确认。
2. 删除数字员工会一并删除数字员工拥有的计划、任务、会话、工件（Artifacts）。
3. 删除时如果当前有任务正在执行，不需要中断任务，但不再展示任务工件。
4. 需要通过定时任务检查已删除的数字员工残留的任务、会话和工件。

### 四、数字员工详情

#### 4.1 数字员工框架

1. 数字员工主要包含声明式定义以及运维两类能力：

| 分类 | 能力 | 使用角色 | 用途 |
| --- | --- | --- | --- |
| 定义 | 数字员工框架 | 管理员 | 数字员工的职责描述 |
| 定义 | 技能 | 管理员 | 数字员工执行任务所需的流程和脚本 |
| 定义 | 知识 | 管理员 | 数字员工完成任务需要的一般领域知识 |
| 运维 | 计划 | 普通用户 | 数字员工的任务计划，包括：定时任务、周期任务、条件任务 |
| 运维 | 会话 | 管理员 | 数字员工的不同会话 session |

#### 4.2 数字员工的设定

1. 数字员工的设定以自然语言对数字员工的职责进行描述。
2. 使用 Markdown 编辑器来进行设定。
3. 数字员工的设定可以作为类似 Skill 的元数据，以便使当前数字员工可以被其他数字员工调用。

#### 4.3 数字员工的知识

1. 知识是数字员工执行任务过程中需要了解的背景信息、业务规则。
2. 可以通过业务知识网络（BKN）来选择该数字员工需要具备的知识，知识可以是结构化或非结构化数据。
3. 在没有合适知识的情况下，可以通过自然语言描述来调用 “BKN 智能体”动态补充业务知识网络并连接数据。
4. 在执行任务时，实际使用的知识来自于：配置到数字员工的知识 + 通过 BKN 动态检索到的知识。
5. 使用图（Graph）的形式来展示知识及知识之间的关系。

#### 4.4 数字员工的技能

1、可以通过业务知识网络（BKN）来选择该数字员工适用的技能，这组技能作为数字员工执行任务时的默认技能，以确保任务执行的可靠性。
2、“技能”节点下挂载已经配置的技能子节点，技能子节点以技能名称命名。点击技能子节点展示：
  - 技能名称
  - 技能描述
  - 脚本（可选）。点击脚本右侧展开抽屉，抽屉以 Markdown 代码块的形式展示脚本内容（通常为 Python 代码）
  - 对话框，用户可以通过自然语言进一步测试技能。
3、在执行任务时，实际使用的技能来自于：配置到数字员工的技能 + 通过 BKN  动态检索到的 Skill。

#### 4.4.1 创建技能
1. 用户可以在“技能”节点通过自然语言对话来调用 “技能智能体”  创建新技能。
2. 创建技能分为以下几步：
  - 1. 生成技能：由“技能智能体” 根据用户意图编写技能描述（Markdown），并在需要时编写技能脚本（Python）
  - 2. 预览技能：技能生成完成后，用户通过与“技能智能体”直接对话来下达技能调用指令以测试技能效果。
  - 3. 发布技能：用户可以一键发布技能，发布技能时，需要填写版本号和发布描述：
    - 版本：遵循语义版本规则，格式为：v.0.0.1。版本号自动递增
    - 描述：必填，不超过 400 个字
3. 技能发布后，在“技能”节点下头部位置插入新增的技能子节点。
4. 发布的技能可以后续被其他数字员工检索并使用。

#### 4.5 管理数字员工的计划和任务

1. 计划是由用户发起的、需要数字员工在满足一定时间条件或逻辑条件下自动执行的待办任务。
2. 计划包含以下属性：
| 属性 | 是否必须 | 说明 |
| --- | --- | --- |
| 名称 | 是 |  |
| 执行条件 | 是 | 可选：<br>- 定时执行<br>- 周期执行<br>- 条件执行 |
3. 计划的本质也是技能，在满足设定的执行条件时触发执行。
4. 计划支持三种执行模式：
  - 定时任务：设定一个一次性的定时任务，到达指定时间后执行，不可重复。
  - 周期任务：设定周期性的定时任务，到达指定时间后执行，可重复。
  - 条件任务：设定监控条件，例如：监听会话消息。
5. 有两种创建计划的路径：
  - 用户可以在“计划”节点通过自然语言创建数字员工的计划。
  - 用户可以在即时通讯工具中 @数字员工 创建计划。

#### 4.5.1 创建计划

1. 用户可以在“计划”节点通过自然语言向数字员工发起创建计划的指令。
2. 创建计划需要经过以下步骤：
  - 1. 生成计划：数字员工根据用户意图，创建计划脚本和 CronJob。
  - 2. 预览计划：用户通过与数字员工对话来下达立即执行的指令以测试计划执行效果。
  - 3. 发布计划：用户确认后，可以一键发布计划。发布后在“计划”节点下头部位置插入新增的计划子节点。
3. 计划子节点展示计划名称和创建的对话，用户可以直接修改计划名称。
4. 计划子节点保留生成计划时的完整对话过程，用户可以继续对话以进一步调整计划执行模式或计划内容。

#### 4.5.2  执行计划

1. 当满足计划条件触发计划执行时， 在计划子节点下再插入一条任务实例。
2. 任务实例包含：
  - 执行结果：成功 / 失败
  - 数字员工执行任务的思考、推理、调用技能的消息流。
  - 任务完成后的工件（Artifacts）。
3. 查看任务工件时，在右侧展开抽屉渲染工件。根据工件类型使用不同的组件进行渲染，包括但不限于：代码、PDF、HTML
4. 支持在任务的消息流中与数字员工进行对话进行进一步追问。

#### 4.6 通道

1. 用户可以在“通道”节点配置如何将数字员工集成到即时通讯工具，支持：飞书、钉钉、企业微信：
| 工具 | 配置项 | Channel 插件 |
| -- | -- | -- |
| 飞书 | - App ID<br>- App Secret |内置 |
| 钉钉 | - Client ID<br>- Client Secret | 内置 |
2. 每个数字员工可以集成一个通道。
